**Module 2 – Assignment & Lab Overview**

** What Module 2 Focused On**
Module 2 introduced you to the *practical workflow* of running Python code in Google Colab and managing your work professionally through GitHub. While Module 1 focused on foundational concepts, Module 2 shifted toward **hands‑on execution, documentation, and industry‑standard organization**.

The module had two major components:

- **A Lab (Lab 1)**  
- **An Assignment (Assignment 2)**  

Both were designed to reinforce the same core skills:  
**running Python code, validating outputs, exporting your work, and organizing your repository.**

** Lab 1 – What It Was About**
The lab guided you through the *technical workflow* of using Google Colab as your coding environment. The main goals were:

**1. Running and validating Python code**
You practiced:
- Executing all cells  
- Checking that outputs were correct  
- Troubleshooting errors  
- Understanding how Python behaves in a notebook environment  

**2. Preparing a clean, professional notebook**
The lab emphasized:
- Running the entire notebook from top to bottom  
- Clearing unnecessary or overwhelming outputs  
- Ensuring reproducibility  

This mirrors real‑world expectations in data science and machine learning.

**3. Exporting your notebook to PDF**
You learned how to:
- Use **File → Print → Save as PDF** (or **Ctrl+P**)  
- Verify that plots, code, and outputs were readable  
- Follow strict naming conventions  

This step teaches you how to present your work clearly — a skill used in academic, research, and industry settings.

** Assignment 2 – What It Was About**
The assignment built on the lab by focusing on **documentation, organization, and professional workflow**.

**1. Understanding Google Colab vs. GitHub**
A major learning objective was recognizing the difference between:
- **Google Colab** → where you *write and run* your code  
- **GitHub** → where you *store, organize, and publish* your work  

This distinction is foundational in machine learning and software development.

**2. Creating a structured GitHub repository**
You were expected to:
- Create a dedicated folder for each assignment  
- Add a detailed README.md  
- Upload your notebook and PDF  
- Follow naming conventions  

This teaches you how to build a repository that looks clean, intentional, and industry‑ready.

**3. Explaining your work**
Your README needed to describe:
- The problem you were solving  
- Your approach and methods  
- Your results and what they mean  

This mirrors how professionals communicate their work in real projects.

**The Purpose of Module 2**
Module 2 wasn’t just about writing Python code. It was about learning the **full workflow**:

| Skill | Why It Matters |
|-------|----------------|
| Running code in Colab | Core skill for ML and data analysis |
| Validating outputs | Ensures accuracy and reproducibility |
| Exporting to PDF | Teaches professional presentation |
| Organizing GitHub repos | Industry standard for collaboration |
| Writing READMEs | Communicates your work clearly |
| Understanding tools | Builds confidence and independence |

By the end of Module 2, you were not just completing tasks, you were learning how to think and work like someone preparing for real machine learning or data science projects.
